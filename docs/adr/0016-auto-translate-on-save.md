# ADR-0016: Auto-translate on save — no manual translation step

**Status:** Accepted

Refines [ADR-0003](./0003-jsonb-i18n-columns.md) (jsonb i18n columns) and revises its re-translate-protection clause. Interacts with [ADR-0014](./0014-review-original-language.md) (review original language) and [ADR-0015](./0015-poi-rich-text-detail.md) (POI rich-text detail).

## Context

Translating dynamic content was a manual, multi-click ritual. To translate a POI the owner wrote the Dutch source, clicked **"Automatisch vertalen"** to open `TranslateDialog`, clicked **"Vertalen"** to fetch machine output, optionally edited the EN/FR/DE textareas, clicked the dialog's **"Opslaan"** to stage the result back into the form, then clicked the form's own **"Opslaan"** to actually persist — four to five clicks across two save buttons, with the dialog acting as a staging area the form's save then wrote. Gallery alt text had the same friction through a separate bespoke path (`translateAltTextAction` → `saveAltTextAction`), and reviews forced a "save the review first, _then_ you can translate" two-step.

The whole apparatus existed to support **hand-editing machine output**, which `_source` then protected from re-translation (ADR-0003). The client has decided they do not want to review or alter translated output — they want correct machine translations to appear with no human step. That removes the dialog's reason to exist and, with it, the human-protection guarantee.

The machine-translation engine itself is already isolated and reusable (`translateToAllLocales`, `translateHtmlToAllLocales`, `translateReviewBody`, and the `lexical/bridge.ts` HTML bridge). Only the _trigger_ and _orchestration_ are in question. The repo has no async/queue/cron infrastructure by deliberate design (ADR-0004/0005 favour lazy-on-read), so "background" execution would be greenfield — and risky, since a Vercel function can be killed the moment it returns its response.

## Decision

Translation becomes an automatic, synchronous side-effect of the existing save actions. The dialog and every hand-edit affordance are removed.

- **Synchronous on save, inside the existing write action.** The save action's flow becomes: load existing row → diff sources → translate the changed fields → write all locales + `_source` → `revalidate`. There is no fire-and-forget and no lazy-on-read. Because translation completes before the action's existing `revalidatePath`/`updateTag`, complete translated content publishes in a single revalidation with no second write and no window where other locales serve stale content. Write volume is one owner editing occasionally, so the few-second worst-case save is acceptable; the Vercel 300s timeout is never approached.
- **Dirty-check per field against the stored source slot.** A field is re-translated only when its source changed — for authored content the `nl` slot, for the rich `detail` the serialized `nl` EditorState JSON, for reviews `original_body`. Saves that touch no source text (e.g. toggling `published`) translate nothing and stay instant, and Google quota is not spent re-translating unchanged copy. On _create_ there is nothing to diff, so every populated field is translated.
- **Self-healing gap-fill.** The re-translate trigger is "source changed **OR** (source is non-empty AND the target locale is empty)". A transient translation failure therefore heals on the next save of that entity, with no queue and no manual retry button. Empty-source → empty-target is correct and never triggers.
- **`_source` is kept; the human-protection guard is deleted.** The columns remain (no migration) and record provenance only: the source slot is `human`, every target locale is `machine`. Machine output is now authoritative — a source change overwrites **all** target locales unconditionally. The previous "re-translate never overwrites a `human` slot" rule (ADR-0003) is gone, because hand-editing is gone. Pre-existing legacy `human` translations survive only while their source is unchanged (a side effect of the dirty-check) and are replaced with machine output the moment that source is next edited.
- **Failures degrade, never block.** The per-field × per-locale calls run in one parallel `Promise.allSettled`. The source always persists, so the owner never loses work and is never blocked by Google downtime; locales that failed are left as-is and surfaced as a non-blocking warning. The public fallback chain (`field[locale] ?? field.nl`, and the longer review chain in `review-i18n.ts`) already renders Dutch / the verbatim original for any behind locale, so a failure shows Dutch, never a blank.
- **Reviews: source language is always auto-detected.** The manual original-language selector is removed. `translateReviewBody` is called on every review save with the `"und"` sentinel; Google detects the language and the detected code is persisted to `original_locale` (so the public "Vertaald uit het …" marker keeps working). `"und"` is now an internal "please detect" signal, never a stored state. The review dirty-check key collapses to `original_body`.
- **UI: dialog removed, replaced by passive feedback.** `TranslateDialog`, every "Automatisch vertalen" trigger, and all editable EN/FR/DE textareas are deleted. The Save button gains a labelled pending state ("Opslaan en vertalen…"), a partial-failure warning names any behind locale, and a **read-only** per-locale indicator (`NL ✓ · EN ✓ · FR ⚠ · DE ✓`, derived purely from which slots are populated — no new storage, no buttons) lets the owner notice a gap and re-save to heal it.
- **No schema change, no backfill.** `_source` columns stay; existing content has no translation gaps, so no backfill script is needed.

## Considered alternatives

- **Fire-and-forget after response** (`after()` / `waitUntil`): the literal reading of "in the background". Rejected — silent failures, no retry, a required second DB write + second revalidation, and a window where other locales serve Dutch-only. It introduces the exact async-failure surface ADR-0004/0005 avoid, for a save only the single owner ever waits on.
- **Lazy-on-read** (the house pattern): store Dutch only, translate the first time a visitor hits a missing locale. Rejected — it puts Google's latency and failure risk on the public render path and doesn't match "on save".
- **Durable queue / job table:** robust but reintroduces async infrastructure the repo deliberately has none of, to solve a throughput problem that does not exist at one-owner write volume.
- **Keep an optional review source-language override:** rejected in favour of pure auto-detect per the client's steer; the trade-off is that a misdetected very-short review ("Top!") is no longer correctable, accepted as rare.

## Consequences

- **Dead code is removed**, not added: `TranslateDialog`; `translateTextAction`, `translateReviewTextAction`, `translateReviewAction`; `translatePoiDetailAction` (`detail-translate-action.ts`); `translateAltTextAction`; and the dialog mount points plus `onTranslated` / `onLocaleEdited` plumbing in `poi-client`, `content-client`, `review-form`, and `gallery-client`. The forms no longer carry EN/FR/DE staging values — only the source. `inferSource` / `buildDetailSource` and the gallery's hardcoded `_source` collapse to "source = human, targets = machine".
- **The engine is untouched.** `translateToAllLocales`, `translateHtmlToAllLocales`, `translateReviewBody`, and `lexical/bridge.ts` keep their signatures; only their callers move into the save actions. The POI slug still derives from the English title — translation now simply runs before slug derivation within the same save.
- **The save action is the single writer** for content _and_ its translations, deepening the existing single-writer invariant (the bug class ADR-0014 closed for reviews now holds for every translatable field).
- **The "background" the client asked for is delivered as "invisible/automatic", not "asynchronous".** The owner does nothing and sees nothing but a brief pending Save — at the cost of that Save being a few seconds on a POI with a long rich-text `detail` (three fields × three locales, the `detail` going through the `happy-dom` bridge), mitigated by firing the whole batch in parallel so wall-clock ≈ the slowest single call.
- **Re-introducing translation review** (a "needs approval" queue, per-locale editing) is the trigger to revisit this ADR; the `_source` provenance columns are deliberately retained so that door is not bricked shut.
